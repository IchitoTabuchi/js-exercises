# 指定のパス文字列を引数にとり、ファイルなら`file`、ディレクトリなら`directory`を返す`checkEntry`関数を`fs.stats`を利用して実装しなさい。なお、可能なら`file`, `directory`以外を返すパターンの必要性を考えてそれも関数内に実装しなさい。

## 実装内容

### file

ファイル

### directory

ディレクトリ

### blockDevice

Linux のブロックデバイス用特殊ファイルで、バッファを使ってハードウェアデバイスにアクセスできる。
ブロックデバイスとは、データをブロック単位で読み書きする入出力デバイスのこと。主にハードディスクやSSDなどのストレージデバイスが該当する。

### characterDevice

Linux のキャラクターデバイス用特殊ファイルで、ブロックデバイス以外のデバイスファイルを扱う。データを1バイトずつ読み書きする入出力デバイスのこと。主にキーボードやマウスなどの入力デバイスが該当する。

### fifo（First In First Out）

Linux の fifo 用特殊ファイルで、書き込んだデータをその順に読み込むことができる。fifoとは、データが最初に入った順番で出力される入出力方式のこと。主にパイプなどの通信手段が該当する。

### socket

Linux のソケット用特殊ファイルで、異なるプロセス間で通信を行うために用いられる。ソケットとは、ネットワーク通信を行うためのエンドポイントのこと。主にTCP/IP通信などや、ログ送信などが該当する。

### 補足

`fs.stats` では、シンボリックリンク自体の情報を取得することができないため、シンボリックリンクを判別することはできない。シンボリックリンクを判別するには、`fs.lstat` を使用する必要がある。
