# 問題 15.11-15.18

Slack や O365 の API は機密情報をヘッダーで指定する。パスやクエリでないのは何故かセキュリティの観点で記載しなさい。

---

## 解答

SlackやO365などのAPIが機密情報（APIキーやアクセストークンなど）をヘッダーで指定するのは、以下のセキュリティ上の理由による。

### パスやクエリパラメータの問題点

- URLのパスやクエリパラメータは、Webサーバーのアクセスログに記録されることで機密情報が漏洩する可能性がある。
- URLはブラウザの履歴に保存されるため、ブックマークに保存されたり、リファラとして他のサイトに送信されたり、プロキシやCDNのキャッシュに保存されたりすることで機密情報が漏洩する可能性がある。

### ヘッダーの利点

- HTTPヘッダーは一部のヘッダーを除いて通常アクセスログに記録されないため、機密情報の露出リスクを低減できる。
- ヘッダーはブラウザの履歴に保存されないため、リファラとして漏洩するリスクがない。
- URL共有を行ってもヘッダーは相手から見えない。
